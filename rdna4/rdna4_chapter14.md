# 第14章：导出：位置、颜色/MRT

"导出"（Export）是指将数据从 VGPR 复制到导出缓冲区（位置缓冲区、颜色缓冲区或深度缓冲区）之一的操作。导出使用 EXEC 掩码，仅输出被启用的像素或顶点。着色器对每个目标只能导出一次。像素着色器的最后一次导出，或顶点着色器的最后一次位置导出，必须标记"done"——表示着色器已完成像素着色器导出或顶点位置导出。这使得数值可分别被渲染后端和图元汇编器消费。

导出可以每元素传输 32 位或 16 位数据。16 位导出以成对形式出现：从一个存储了两个 16 位值的 VGPR 中传输 32 位数据。导出指令本身不区分两者，只是按通道移动 32 位数据。16 位导出是着色程序（负责转换和打包 16 位数据）与接收硬件（通过配置寄存器声明导出数据类型）之间的协议。16 位数据被打包到 VGPR 中，第一个分量位于低 16 位。

**指令字段**

| 字段 | 大小（位） | 描述 |
|------|-----------|------|
| Done | 1 | 表示这是着色器的最后一次导出。仅用于像素、位置和图元数据。图元导出时必须置位 |
| Target | 6 | 导出目标：<br>0-7：MRT 0-7<br>8：深度（Z）<br>12-16：位置 0-4（Pos4 用于立体渲染）<br>20：NGG 图元数据（连接性数据）<br>21：双源混合左<br>22：双源混合右 |
| EN | 4 | 16 位分量：导出半 DWORD 使能。有效值为：0x0、1、3<br>[0] 使能 VSRC0：来自一个 VGPR 的 R、G（R 在低位，G 在高位）<br>[1] 使能 VSRC1：来自一个 VGPR 的 B、A（B 在低位，A 在高位）<br>32 位分量：[0-3] 分别为 VSRC0-3 的使能位 |
| VSRC0 | 8 | 读取数据的 VGPR（位置：vsrc0=X，1=Y，2=Z，3=W；MRT：vsrc0=R，1=G，2=B，3=A） |
| VSRC1 | 8 | 同上 |
| VSRC2 | 8 | 同上 |
| VSRC3 | 8 | 同上 |
| ROW_EN | 1 | 0=正常模式；1=使用 M0 提供网格着色器 POS 和 PRIM 导出的行号 |
| (M0) | 8 | 网格着色器 POS 和 PRIM 导出的行号 |

**32 位分量分配**

| EN[0] | EN[1] | EN[2] | EN[3] | VSRC0 | VSRC1 | VSRC2 | VSRC3 |
|-------|-------|-------|-------|-------|-------|-------|-------|
| 红/X/… | 绿/Y/… | 蓝/Z/… | Alpha/W/… | | | | |

| EN[0] | EN[1] | EN[2]、EN[3] | VSRC0 | VSRC1 | （忽略） |
|-------|-------|------------|-------|-------|--------|
| {绿, 红} / {y, x} | {Alpha, 蓝} / {w, z} | 未使用 | | | |

## 14.1 像素着色器导出

导出指令将颜色数据复制到 MRT（多重渲染目标）。数据最多有四个分量（R、G、B、A）。导出指令还可选择输出深度（Z）数据。

每个像素着色器必须至少有一条导出指令。执行的最后一条导出指令必须将 DONE 位置为 1。

EXEC 掩码适用于所有导出操作。只有对应 EXEC 位为 1 的像素才将数据导出到输出缓冲区。

每个导出目标只能被导出一次。

着色程序负责将数据从 32 位转换为 16 位（用于 16 位导出）。

着色程序负责 alpha 测试。如果着色器可以丢弃像素，必须在第一次导出之前完成判断。每次像素着色器导出必须使用相同的像素有效掩码（EXEC），且该掩码必须是最终掩码值（后续导出不能使用不同的掩码值）。

像素导出在为 wave 分配了导出缓冲区空间后才会执行。若 STATUS.SKIP_EXPORT==1，则着色程序忽略导出指令。

**OREO - 不透明像素数据随机顺序导出**

OREO 允许不透明表面乱序导出。若没有此机制，所有导出必须在 wave 之间按顺序进行。OREO 具有冲突检测机制（检测写入同一像素的 wave），并强制执行顺序约束。若某 wave 存在"冲突"，该 wave 的导出在冲突存在期间可能以较低速率进行。无需软件支持。

所有可能影响采样掩码的数据必须在着色器的第一次导出中发送。若要导出深度，必须首先导出深度。若要导出 alpha-to-mask，MRT0 必须首先导出，除非同时启用了深度导出，此时 MRT0 的 alpha 值必须写入深度导出的 alpha 值。若同时启用了 alpha-to-mask 和 coverage-to-mask，则深度导出的 alpha 值应设置为 alpha-to-mask 值（MRT0 的 alpha）与 coverage-to-mask 值（本应写入深度导出的 alpha）两者的最小值。

**像素着色器双源混合**

在此模式下，交替通道（线程）分别持有 MRT0 和 MRT1 的数据，而非所有线程都对应同一个 MRT。完成一次双源混合导出需要两条指令。要求对目标 21 和 22 的导出必须连续执行，中间不能插入其他类型的导出。

| 导出目标 | EXEC 掩码 | 导出的 MRT | 通道 0 | 通道 1 | 通道 2 |
|---------|----------|----------|--------|--------|--------|
| 21 | `exec_mask = (exec_mask & 0x5555_5555) \| ((exec_mask << 1) & 0xAAAA_AAAA)` | 0 | Pix0 MRT0 | Pix0 MRT1 | Pix2 MRT0 |
| 22 | `exec_mask = (exec_mask & 0xAAAA_AAAA) \| ((exec_mask >> 1) & 0x5555_5555)` | 1 | Pix1 MRT0 | Pix1 MRT1 | Pix3 MRT0 |

## 14.2 图元着色器导出（来自 GS 着色器阶段）

GS 着色器使用导出指令输出顶点位置数据，并使用内存存储操作输出顶点参数数据。这些数据被传递给后续的像素着色器。

每个顶点着色器必须至少向 POS0 目标输出一个位置向量（x、y、z；w 为可选）。最后一次位置导出必须将 DONE 位置为 1。

## 14.3 依赖性检查

导出指令由硬件分两个阶段执行。第一阶段：指令被选中执行，EXPCNT 加 1，此时 wave 已发出导出数据的请求，但数据尚未实际导出。第二阶段：实际导出时，读取 EXEC 掩码和 VGPR 数据并完成导出，EXPCNT 减 1。

使用 `S_WAIT_EXPCNT` 可防止着色程序在导出操作完成之前覆盖 EXEC 或 VGPR 中待导出的数据。

多条导出指令可同时处于未完成状态。相同类型的导出（例如：位置导出）按顺序完成，但不同类型的导出可乱序完成。若 STATUS 寄存器的 SKIP_EXPORT 位为 1，硬件将所有 EXPORT 指令视为 NOP 处理。
