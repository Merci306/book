# 第四章：着色器指令集

本章描述着色器指令集。指令分为以下几组：

- **程序流程控制**
- **标量 ALU**
- **从常量缓存读取的标量内存**
- **向量 ALU 与参数插值**
- **向量内存读/写**：缓冲区、图像；Flat、全局和暂存；LDS
- **导出**：位置、图元、颜色/MRT/Z

指令以各种微码格式编码。这些格式由一组"编码"位定义，这些位确定指令系列以及指令中其余位的含义。并非每条指令都使用其编码中的每个字段。可以将 SGPR 指定为源或目标的字段在未使用时通常设置为 NULL；其他字段通常设置为零。

---

## 4.1 通用指令字段

**内联常量（inline constant）**：硬连接到处理器中的常量，用于代替源参数指定。例如 1.0、-0.5、32 等。

- 浮点常量适用于单精度、双精度和 16 位浮点指令；在非浮点指令中使用时，数据不被转换（保持为浮点格式）
- 浮点常量根据源操作数的大小编码。对于 16 位操作（打包和非打包），浮点常量被视为零扩展的 32 位数据（低位为 16 位浮点，高位为零）
- 用于 32 位或更小操作数的整数常量被视为 32 位有符号整数；对于 64 位源，整数常量进行符号扩展

**32 位字面常量（32-bit literal constant）**：紧接在 32 位或 64 位指令之后的指令流中的 32 位常量。

- 用于 64 位有符号整数操作时，符号扩展到 64 位
- 用于无符号 64 位整数操作（和 64 位二进制操作）时，零扩展
- 用于双精度浮点操作时，32 位字面量为最高有效位，低有效位为零
- 其他操作（32 位或更小，或打包数学运算）将其视为 32 位数据，通常使用低有效位

### 4.1.1 缓存控制：SCOPE 与 Temporal-Hint

标量和向量内存指令包含两个字段，用于控制缓存作用域（哪级缓存保存数据）和时间提示（建议数据可能多快被复用）。

这些位也控制内存原子操作，指示它们是否返回操作前的值。

ISA SCOPE 位对应 4 个缓存级别，指示给定缓存是否可以在本地完成操作，或是否需要将操作转发到缓存层次结构的下一级，以在所需作用域（一致性域）完成操作。

| # | 名称 | 含义 |
|---|------|------|
| 0 | CU | 计算单元（工作组）作用域——在 CU（工作组，但不是 WGP）中所有 VMEM 线程之间保持一致 |
| 1 | SE | 着色器引擎——在共享 SE 缓存的所有客户端（线程）之间保持一致 |
| 2 | DEV | 设备——在同一设备上的所有线程之间保持一致 |
| 3 | SYS | 系统 |

**缓存层次结构中每级的规则：**

- 若 ISA.scope <= 缓存作用域：
  - 读取可命中缓存
  - 写入缓存，事务从此级缓存确认
  - 读-修改-写操作可在此缓存本地发生
- 否则：必须将操作"转发"到下一个更大作用域的缓存
  - 转发 = 加载不能命中此缓存，存储必须传播到下一级更高缓存（但也更新此级缓存状态）

**释放（Release）与获取（Acquire）**

- **释放**：生产/写入数据的线程可执行释放操作，将所有先前写入提升到释放指定的更高作用域。硬件 ISA 实现使用带作用域的"WB"（写回）操作
  - 若 WB 操作且 ISA.SCOPE > 缓存作用域：将所有脏块刷新到下一级层次结构
  - 若 WB 操作且 ISA.SCOPE <= 缓存作用域：NOP——不需要刷新

- **获取**：消费/读取数据的线程可执行获取操作，将所有后续读取提升到获取指定的作用域。硬件 ISA 实现使用带作用域的"INV"（无效化）操作
  - 若 INV 操作且 ISA.SCOPE > 缓存作用域：无效化所有块，使后续读取从下一级缓存重新读取
  - 若 INV 操作且 ISA.SCOPE <= 缓存作用域：NOP——不需要无效化

**时间提示（TH）** ISA 位提供预期数据复用的指示，用于缓存层次结构中数据保留的优先级。存储到特定作用域时，从该作用域返回"完成"（递减 STOREcnt）。

**表 15. 读-修改-写原子操作的 TH 策略**

对于 RMW 原子操作，TH 字段分为 3 个独立字段：

| 字段 | 含义 |
|------|------|
| TH[0] | 0：非返回原子 RMW 操作（无数据返回到 VGPR）；1：返回原子 RMW 操作（数据返回到 VGPR） |
| TH[1] | 0："RT"——常规（默认）时态，预期复用；1："NT"——非时态，不预期复用（也称"流式"） |
| TH[2] | 级联（延迟作用域）；0：常规原子操作，不延迟作用域（TH[0]==1 的原子不延迟）；1：级联原子操作，若 TH[0]==0 则延迟作用域 |

- 原子操作可选择在执行原子操作前返回内存中的值或不返回
- 级联原子操作用于直方图（非返回）类型原子操作，完整的指定作用域在后续匹配或更高作用域的释放"同步/栅栏/原子操作"发生之前不会实现

**SMEM 加载操作的 SCOPE 字段：**

SMEM 操作具有与 VMEM 加载操作编码相同的 2 位 SCOPE 字段和相同的定义。注意"CU"作用域只意味着与同一 CU 上的其他标量线程保持一致，而不与向量线程保持一致。对于同步同一 CU 的标量和向量内存线程，必须使用 scope > 0。

SMEM 也支持 2 位 TH 字段（与 VMEM TH[1:0] 枚举相同，0 到 3）；SMEM 不支持独立的近缓存和远缓存时间提示，提示在层次结构的所有缓存中统一应用。

作用域为 CU 的 SMEM 加载与 VMEM 存储/原子不保持一致。
