## 前言

### 关于本文档

本文档描述了 AMD "RDNA4" 世代设备的当前环境、组织结构及程序状态，详细介绍了该处理器系列原生的指令集及微码格式，供程序员和编译器开发者参考。

文档规定了各指令（包括每类指令的格式）以及相关的程序状态（包括程序状态与指令之间的交互方式）。某些指令字段之间存在相互依赖关系，并非所有字段的所有取值组合都是合法的，本文档给出了合法的组合方式。

本文档的主要目的：

1. 规定语言构造及其行为，包括每类指令在文本语法和二进制格式两方面的组织形式。
2. 为编译器开发者提供指令操作参考，以最大化处理器性能。

### 表 1　符号约定

| 符号 | 含义 |
|------|------|
| `*` | 代码格式、参数或指令名称中任意数量的字母数字字符。 |
| `[1,2)` | 包含左端值（1），但**不包含**右端值（2）的范围。 |
| `[1,2]` | 同时包含左端值和右端值的范围。 |
| `{x \| y}` 或 `{x, y}` | 列出的多个选项之一。例如 `S_AND_{B32, B64}` 定义了两条合法指令：`S_AND_B32` 和 `S_AND_B64`。 |
| `0.0` | 浮点值。 |
| `1011b` / `'b0010` / `32'b0010` | 二进制值。`32'b0010` 表示 32 位二进制值。二进制值可包含下划线以提高可读性，解析时忽略下划线。若位数少于 32 位，高位默认为零。 |
| `0x1A` / `'h123` / `24'h01A3B4` | 十六进制值。`24'h01A3B4` 表示 24 位十六进制值。 |
| `7:4` 或 `[7:4]` | 位范围，从第 7 位到第 4 位（含两端）。高位在前。 |
| *斜体词或短语* | 对理解基于 GPU 的计算至关重要的术语或概念的首次出现。 |

## 1.2　硬件概述

RDNA4 设备包含**数据并行处理器阵列**、**命令处理器**、**内存控制器**及其他逻辑。命令处理器读取主机写入系统内存地址空间中内存映射寄存器的命令，并在命令完成后向主机发送硬件中断。内存控制器可直接访问所有设备内存及主机指定的系统内存区域，具备 DMA 控制器的功能。

在 RDNA4 环境中，一个完整的应用由两部分组成：

- 运行在**主机处理器**上的程序。
- 运行在 RDNA4 处理器上的程序，称为**着色器程序**或**内核**。

主机上运行的驱动程序负责控制 RDNA4 程序，具体包括：

- 设置内部基地址及其他配置寄存器。
- 指定 RDNA4 处理器要操作的数据域。
- 无效化并刷新 RDNA4 处理器上的缓存。
- 触发 RDNA4 处理器开始执行程序。

### 1.2.1　工作组处理器（WGP）

处理器阵列是 RDNA4 处理器的核心，由一组相互独立的**工作组处理器（WGP）**流水线组成，各流水线并行处理浮点或整数数据流。

WGP 流水线可执行数据计算，也可通过内存控制器向内存读写数据。计算操作和内存写出均支持条件执行。

当 WGP 流水线收到请求后，会：

1. 从内存加载指令和数据。
2. 开始执行，直至内核结束。

指令预取**完全由硬件完成**，软件无需介入。内核可将数据从片外内存加载到片上 GPR 和缓存中。

RDNA4 设备通过以下方式隐藏内存延迟：

- 同时追踪处于不同执行阶段的数百个工作项。
- 将计算操作与内存访问操作重叠执行。

### 1.2.2　缓存层次结构

内存系统划分为多个层次。每个工作项可访问的存储层次如下：

```
工作项
  │
  ▼
VGPR / SGPR（片上寄存器，VGPR 每工作项私有 / SGPR 每 wave 共享）
  │
  ▼
LDS（本地数据共享，每 WGP 128 kB，工作组内共享）
  │
  ▼
L0 缓存（每 WGP，只读指令缓存与标量缓存）
  │
  ▼
GL1 缓存（写合并缓存，每着色器阵列）
  │
  ▼
L2 缓存（读写缓存，含原子单元，每设备）
  │
  ▼
设备内存（VRAM）
```

#### 1.2.2.1　本地数据共享（LDS）

同一工作组内的工作项可通过 **LDS** 共享数据。

| 属性 | 规格 |
|------|------|
| 每 WGP 容量 | 128 kB |
| 存储组数 | 64 组 |
| 每组条目数 | 512 × 4 字节 |
| 整数原子单元数 | 64 个 |
| 每工作组最大分配 | 64 kB |

LDS 的用途包括：

- 工作组或 wave 内工作项之间的低延迟数据共享。
- 可预测重用数据的软件缓存。
- 工作组内工作项的数据交换机制。
- 高效协作访问片外内存。

### 1.2.3　设备内存

RDNA4 设备为 WGP 中的处理单元（PE）提供多种访问片外内存的方式。

**主读路径：**

- 多通道 **L2 缓存** → 只读 **GL1 缓存** → 每 WGP 的 **L0 缓存**。
- **GL1**：写合并缓存，汇聚离散写操作，为内存提供高效的访问模式。
- **L2**：读写缓存，含原子单元，使每个 PE 能完成无序原子访问并返回初始值。

特定的**无缓存加载指令**可强制在执行时直接从设备内存读取数据，绕过缓存。

**指令缓存与常量缓存**（只读）：

- **指令缓存**：向每个 SIMD 提供着色器指令。
- **常量缓存**：提供对标量常量的访问。

**原子操作：**

每个 PE 需提供：
- 原子操作作用的目标地址。
- 原子操作使用的数据。
- 用于存储操作前值的返回地址。

每个存储或原子操作可在以下情况收到缓存系统的**确认信号**：

- 写操作在请求的缓存范围内完成时；或
- 带返回值的原子操作从内存获取"操作前值"时。

该确认机制使 PE 能够实现**内存屏障（fence）**：在完成后续写操作之前，确保所有写操作已提交到内存，从而维护松散一致性模型。同一 PE 向同一内存通道的离散写操作保持顺序。
