# 第 2 章　有符号整数的表示（Representing Signed Numbers）

**《计算机算术：算法与硬件设计》（Computer Arithmetic: Algorithms and Hardware Designs）第 2 版**  
Behrooz Parhami，Oxford University Press，2010

> 本章介绍有符号整数的多种表示方法：符号数值、偏置、补码（基数补码/数字补码）、2 的补码、1 的补码，以及有符号数字（GSD）。每种表示在范围、运算代价、特殊情况处理上各有取舍。

---

## 2.1 符号数值表示（Signed-Magnitude Representation）

### 基本结构

最直观的有符号整数表示：用最高位（MSB）作**符号位**，其余位表示绝对值（数值部分）。

- 符号位：`0` = 正，`1` = 负
- 数值部分：k−1 位无符号整数
- 表示范围：$[-(2^{k-1}-1),\ +(2^{k-1}-1)]$（**对称**）

### 特点与问题

| 特性 | 说明 |
|------|------|
| **零的两种表示** | `+0`（`00...0`）和 `-0`（`10...0`）都合法，导致判零需要特殊处理 |
| **范围对称** | 正负范围相同，无偏差 |
| **加减法复杂** | 符号不同时需比较绝对值大小，分类处理，电路复杂 |
| **乘除法直观** | 先计算绝对值结果，再异或符号位 |

### 硬件实现（图 2.2）

取反（求负）：仅翻转符号位  
绝对值：若符号位为 1 则输出补码，否则原样输出

符号数值表示在**浮点数尾数**和**BCD 有符号十进制**中仍有应用。

## 2.2 偏置表示（Biased Representation）

### 定义

选定一个偏置量（bias）`B`，将整数 `x` 存储为无符号整数 `x + B`：

$$\text{存储值} = x + B$$

- 表示范围：若使用 k 位无符号整数（范围 `[0, 2^k−1]`），则 `x` 的范围为 `[−B, 2^k−1−B]`
- 典型偏置：`B = 2^{k-1} − 1`（IEEE 754 浮点指数使用）或 `B = 2^{k-1}`

### 应用：IEEE 754 浮点数指数

单精度（32位）浮点数指数使用 8 位偏置表示，偏置 `B = 127`：

| 存储值（8位无符号） | 实际指数 |
|:---:|:---:|
| 0 | −127（特殊：非规格化数） |
| 1 | −126 |
| 127 | 0 |
| 254 | +127 |
| 255 | +128（特殊：∞ 和 NaN） |

### 优点

- **指数比较简单**：直接比较无符号存储值即可比较实际指数大小
- **加法需修正**：两数指数相加时，结果偏置多加了一次，需减去 `B`

## 2.3 补码表示（Complement Representation）

### 一般化补码

给定**补码常数**（complementation constant）`M`，整数 `x` 的补码定义为：

$$\bar{x} = M - x$$

若 `M = r^k`（k 位 r 进制数的模），则加法在模 `M` 意义下自动处理借位/退位，使减法转化为加法。

### 模 M 加法表

对 k 位数：
- 若 `x + y < M`：结果为 `x + y`（无溢出）
- 若 `x + y ≥ M`：丢弃进位，结果为 `x + y − M`

这保证了**减法 = 加补码**，无需单独的减法电路。

### 基数补码（Radix Complement）

取 `M = r^k`：

$$\bar{x} = r^k - x$$

二进制（r=2）时称 **2 的补码（2's complement）**。

### 数字补码（Diminished-Radix Complement）

取 `M = r^k − 1`：

$$\bar{x} = (r^k - 1) - x$$

二进制（r=2）时称 **1 的补码（1's complement）**。

## 2.4 2 的补码（Two's-Complement）

### 表示范围

k 位 2 的补码整数的范围为：

$$[-2^{k-1},\ +2^{k-1}-1]$$

**不对称**：负数多一个（最小值 `−2^{k-1}` 无对应正数）。

| k | 最小值 | 最大值 |
|---|--------|--------|
| 4 | −8 | +7 |
| 8 | −128 | +127 |
| 16 | −32768 | +32767 |
| 32 | −2147483648 | +2147483647 |

### 求负（取补）

$$-x = \bar{x} + 1 \quad (\text{逐位取反再加 1})$$

快捷规则：从低位起，找到第一个 `1`，保留该位及其右侧所有位不变，对左侧所有位取反。

### 符号位的意义

MSB 的权重为 $-2^{k-1}$（**负权重**），其余位权重为正：

$$x = -x_{k-1} \cdot 2^{k-1} + \sum_{i=0}^{k-2} x_i \cdot 2^i$$

### 符号扩展（Sign Extension）

将 k 位 2 的补码扩展为 n 位（n > k）：将符号位复制填充到高位：

$$[x_{k-1}, x_{k-1}, \ldots, x_{k-1}, x_{k-2}, \ldots, x_0]$$

例：4 位 `1101`（= −3）→ 8 位 `11111101`（= −3）

### 2 的补码加减法

```
加法：直接按位相加，丢弃最高进位（模 2^k）
减法：x − y = x + (−y) = x + ȳ + 1
溢出检测：当两个同符号数相加得到异符号结果时发生溢出
          V = C_{k-1} XOR C_k  （进位链中最高两位进位的异或）
```

### 主要优点

- **唯一零表示**：`0` 只有一种表示
- **加减法统一**：只需一个加法器
- **符号扩展简单**：复制符号位
- **计算机主流选择**：现代 CPU 整数运算几乎全用 2 的补码

## 2.5 1 的补码（One's-Complement）

### 表示与范围

$$\bar{x} = (2^k - 1) - x = \underbrace{11\ldots1}_{k} - x$$

即：**逐位取反**。

- 表示范围：$[-(2^{k-1}-1),\ +(2^{k-1}-1)]$（对称）
- 零的两种表示：`00...0`（+0）和 `11...1`（−0）

### 加法：末位循环进位（End-Around Carry）

```
步骤：
1. 将两数按位相加（包括符号位）
2. 若产生最高位进位 C_k，则将该进位加回到最低位
3. 结果即为 1's complement 表示的和
```

例：`(−3) + (−4)` 在 4 位 1's complement：
```
  1100  (−3)
+ 1011  (−4)
------
1 0111  → 进位 1 加回 → 0111 + 1 = 1000 (−7) ✓
```

### 与 2 的补码比较

| 特性 | 1's complement | 2's complement |
|------|:---:|:---:|
| 范围对称 | ✓ | ✗ |
| 唯一零 | ✗ | ✓ |
| 取反电路 | 逐位非门 | 逐位非门 + 加 1 |
| 加法修正 | 末位循环进位 | 无需修正 |
| 现代应用 | 极少（仅网络校验和） | 主流 |

## 2.6 有符号整数的直接与间接运算

### 直接法

设计专门针对某种表示的算术电路（如专用补码加法器）。  
优点：无转换开销；缺点：每种表示需要独立电路。

### 间接法

1. 将有符号表示**转换为无符号**或其他标准格式
2. 执行无符号运算
3. 将结果**转换回**原始表示

常见于不直接支持有符号运算的场合（如某些 DSP 或历史架构）。

## 2.7 有符号数字（Signed Digits）与扩展点表示法

### 有符号数位（Signed Bit Positions）

一般化 2's complement 概念：不仅最高位，**任意位**都可以赋予负权重。

扩展点表示法（Extended Dot Notation）：
- **实心点（posibit）**：正权重位（普通位）
- **空心点（negabit）**：负权重位

例：4 位数 `[1̄, 0, 1, 1]`（最高位为 negabit）表示：
$$-8 + 0 + 2 + 1 = -5$$

### 广义有符号数字（Generalized Signed Digits, GSD）

数字集扩展为 `[-α, β]`，其中 `α, β ≥ 0`，`α + β ≥ r − 1`。

- **非冗余**：`α + β = r − 1`（如标准 2's complement 中最高位为 −1，其余为 [0, r−1]）
- **冗余**：`α + β ≥ r`（允许多种表示，可实现无进位加法，见第 3 章）

### 冗余度指标（Redundancy Index）

$$\rho = \alpha + \beta - (r - 1)$$

- `ρ = 0`：非冗余（唯一表示）
- `ρ > 0`：冗余，允许无进位加法

### 二进制有符号数字（BSD, Signed Binary）

r=2，数字集 `[-1, 1]`（即 `{1̄, 0, 1}`，其中 `1̄` 表示 −1）：

$$\rho = 1 + 1 - (2 - 1) = 1$$

BSD 可实现**无进位加法**（第 3 章核心），但每位需要 2 bit 存储（如 `{00, 01, 11}` 或 `{00, 01, 10}` 编码）。

## 本章小结

```
有符号整数表示
    ├── 符号数值（Signed-Magnitude）
    │   ├── 范围对称 [-(2^{k-1}-1), +(2^{k-1}-1)]
    │   ├── 两个零：+0 和 -0
    │   └── 乘除直观，加减复杂
    ├── 偏置（Biased / Excess-B）
    │   ├── 存储 x+B，比较方便
    │   └── 用于 IEEE 754 指数字段
    ├── 基数补码（Radix Complement）= 2's complement（二进制）
    │   ├── 范围 [-2^{k-1}, 2^{k-1}-1]（不对称）
    │   ├── 唯一零，加减法统一
    │   └── 现代 CPU 主流选择
    ├── 数字补码（Diminished-Radix Complement）= 1's complement（二进制）
    │   ├── 范围对称，两个零
    │   └── 加法需末位循环进位
    └── 有符号数字（GSD）
        ├── 数字集 [-α, β]
        ├── 冗余度 ρ = α+β-(r-1)
        └── BSD（ρ=1）→ 无进位加法（第 3 章）
```

**三种核心表示（设计者必须掌握）**：
1. **2's complement**：整数运算的标准格式
2. **Carry-Save（存储进位）**：加速乘法/除法的冗余格式（第 3 章）
3. **IEEE 754 浮点数**：实数表示（第 17 章）
