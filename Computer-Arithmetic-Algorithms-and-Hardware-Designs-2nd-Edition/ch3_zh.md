# 第 3 章　冗余数制（Redundant Number Systems）

**《计算机算术：算法与硬件设计》（Computer Arithmetic: Algorithms and Hardware Designs）第 2 版**  
Behrooz Parhami，Oxford University Press，2010

> 本章介绍**冗余数制**：通过允许某些数值有多种表示，换取**无进位传播加法**的能力。这是高速乘法、除法和函数计算的基础。核心内容：存储进位格式、广义有符号数字（GSD）分类、无进位加法算法、有限进位加法、数字集转换、支持函数。

---

## 3.1 进位传播问题（The Carry Problem）

### 进位链的代价

标准二进制加法：低位进位必须**串行传播**至高位（最坏情况需经过 k 级门延迟），对 k 位加法器延迟为 $O(\log k)$（超前进位）至 $O(k)$（行波进位）。

这一限制成为高速算术的**性能瓶颈**。

### 四种解决思路

| 方案 | 思路 | 代表技术 |
|------|------|---------|
| **加速进位传播** | 保持标准表示，但用更快电路 | 超前进位（CLA）、选择进位、条件求和 |
| **存储中间进位** | 不立即合并进位，累积到最后再处理 | Carry-Save 加法（§3.2） |
| **冗余数字集** | 允许多种表示，单步加法不产生进位 | 有符号数字（BSD/SD）加法（§3.3–3.4） |
| **非位置制** | 完全规避进位概念 | 余数数制 RNS（第 4 章） |

## 3.2 存储进位格式（Stored-Carry / Carry-Save Format）

### 基本思想

将 k 位数表示为**两个 k 位数之和**（不实际求和），即用 `(S, C)` 对保存"和"与"进位"两个向量：

$$x \equiv S + C$$

数字集：每个"位位置"可存储 `{0, 1, 2}`（即 `S_i + C_i ∈ {0,1,2}`），等价于基数 2、数字集 `[0, 2]`。

冗余度：`ρ = 0 + 2 - (2 - 1) = 1`（满足冗余条件）

### (3;2) 计数器（全加器）

三个输入位相加产生一个（和，进位）对：

```
输入：a, b, c
输出：s = a XOR b XOR c
      c_out = majority(a, b, c) = ab + bc + ca
```

利用 n 个 (3;2) 计数器，可在**一级门延迟**内将 3 个数加为 2 个数（进位保存树/华莱士树）。

### 进位保存加法树（Wallace Tree）

对 m 个数求和：
- 每轮用 (3;2) 计数器将 m 个数压缩为 $\lceil 2m/3 \rceil$ 个数
- 重复直到剩 2 个数
- 最后用一个标准加法器（CLA）合并

压缩速度：$O(\log_{3/2} m)$ 层，总延迟仅 $O(\log m)$（与 k 无关！）

**应用**：加速乘法（部分积求和）是 Carry-Save 最重要的应用场景（第 9–12 章）。

## 3.3 广义有符号数字（Generalized Signed Digits, GSD）

### 定义回顾

基数 `r`，数字集 `[-α, β]`，`α, β ≥ 0`，`α + β ≥ r − 1`。

**冗余度指标**（Redundancy Index）：
$$\rho = \alpha + \beta - (r - 1) \geq 0$$

- `ρ = 0`：非冗余（标准表示，唯一性）
- `ρ ≥ 1`：冗余，可实现无进位传播加法

### GSD 分类树（图 3.6）

```
GSD 数制
├── α = 0（无负数字）→ 标准无符号数（ρ=0）或高基冗余数（ρ>0）
├── β = 0（无正数字）→ 负数字系统（如 negabinary）
└── α,β > 0（混合）
    ├── r = 2（二进制基数）
    │   ├── 数字集 {0,1,2}：存储进位（Carry-Save）
    │   ├── 数字集 {-1,0,1}：BSD（二进制有符号数字，ρ=1）
    │   └── 数字集 {-1,0,1,2} 等：高冗余（ρ=2）
    └── r > 2（高基数）
        ├── 标准（r=10，十进制）
        └── 有符号十进制、混合基数等
```

### BSD 编码

二进制有符号数字（Binary Signed Digit）：r=2，数字集 `{1̄, 0, 1}`（其中 `1̄` = −1）：

每位需要 **2 bit** 存储，常用编码：

| 数字值 | 编码（正/负双轨） | 编码（两位二进制） |
|:---:|:---:|:---:|
| +1 | (1, 0) | 01 |
| 0  | (0, 0) | 00 |
| −1 | (0, 1) | 11 或 10 |

**注**：编码 `11` 和 `10` 均可表示 −1，导致非唯一性（冗余的来源）。

## 3.4 无进位加法算法（Carry-Free Addition）

### 适用条件

对基数 r、冗余度 ρ 的 GSD 数制，可以实现**完全无进位传播**的加法，当且仅当：
- **r > 2** 时，ρ ≥ 1 即可（单步进位消除）
- **r = 2** 时，需要 ρ ≥ 1 且 α ≠ 1 或 β ≠ 1（即不能是标准 BSD `{-1,0,1}`）

**BSD（r=2，ρ=1）的特殊处理**：需要两步（有限进位，见 §3.5）

### 算法思路（以 r=10 冗余十进制为例）

数字集 `[-4, 5]`（或 `[-5, 4]`，ρ=0），输入两个数字 `x_i, y_i ∈ [-4, 5]`：

1. **预移位**：将和 `w_i = x_i + y_i ∈ [-8, 10]` 分解为"输出数字 + 进位"
2. **进位限幅**：确保进位只取 `{-1, 0, 1}`（有限集合，不会链式传播）
3. **单步计算**：`d_i = w_i - r × t_i`，其中 `t_i ∈ {-1, 0, 1}` 为进位

关键性质：由于 ρ ≥ 1，可以选择 `t_i` 使 `d_i ∈ [-α, β]`，进位 `t_i` **仅向左传一位**，不链式传播。

### 二进制有符号数字加法（BSD，r=2）

每个位的加法输入为 `x_i, y_i ∈ {-1, 0, 1}`，和 `w_i = x_i + y_i ∈ {-2, -1, 0, 1, 2}`：

使用**两步过程**（有限进位传播，见 §3.5）将 BSD 加法的进位传播限制到 1 位。

## 3.5 有限进位加法（Limited-Carry Addition）

当 ρ = 1（BSD 等情况）时，无法完全消除进位，但可以将进位传播**限制为固定步数**（有限传播）。

### 三种变体

#### 变体 1：进位估计（Carry Estimation）

在确定精确结果之前，先快速估计每位的进位方向（`+1`、`0` 或 `−1`），再进行修正。

适用于：流水线加法器中，预测下一级进位以减少等待时间。

#### 变体 2：重复进位（Repeated Carries）

分多轮（通常 2 轮）处理进位：
- 第 1 轮：将三个数压缩为两个（Carry-Save 风格）
- 第 2 轮：用有限进位加法合并，保证进位最多传播 1 位

适用于：累加器结构（部分积求和）。

#### 变体 3：并行进位（Parallel Carries）

同时计算多种进位情景（类似超前进位），时间复杂度 $O(1)$，代价是更多硬件。

### BSD 两步无进位加法

对 BSD 加法（数字集 `{-1, 0, 1}`，ρ=1），标准过程：

**步骤 1**（转换为中间冗余）：
- 对每位 `w_i = x_i + y_i ∈ {-2,...,2}` 计算传输数字 `t_i` 和初步数字 `s_i`
- 规则：若 `w_i = 2` 则 `t_i = 1, s_i = 0`；若 `w_i = -2` 则 `t_i = -1, s_i = 0`；否则 `t_i = 0, s_i = w_i`

**步骤 2**（吸收来自右侧的进位）：
- `d_i = s_i + t_{i-1}`（`t_{i-1}` 是来自右侧的传输）
- 由构造可知 `d_i ∈ {-1, 0, 1}`，保证结果合法

**延迟**：两步逻辑，$O(1)$（与操作数位数 k 无关）

## 3.6 数字集转换（Digit-Set Conversions）

### 目的

在不同冗余数制之间转换，或从冗余转换到非冗余（标准表示）。

### 示例 3.1：BSD → 2's complement

将 n 位 BSD 数 `[d_{n-1}, ..., d_0]`（数字 ∈ {-1, 0, 1}）转换为 2's complement：

方法：分离正负数字：
- `p_i = max(d_i, 0)`（正数部分）
- `n_i = max(-d_i, 0)`（负数部分的绝对值）
- 结果 = `P - N`（两个无符号数之差）

用一个超前进位减法器实现。

### 示例 3.2：存储进位（S,C）→ 2's complement

```
直接相加：X = S + C，用一个标准 CLA 加法器实现
延迟：O(log k)
```

### 示例 3.3：冗余十进制 → 标准十进制

数字集 `[-4, 5]` → `[0, 9]`：
- 若 `d_i ∈ [0, 5]`：直接输出，进位 = 0
- 若 `d_i ∈ [6, 9]`（注：此处为类似 BCD）：输出 `d_i - 10`，进位 = 1  
（具体边界取决于数字集选择）

### 示例 3.4：高冗余 → 低冗余

将冗余度 ρ=2 的数制转换为 ρ=1：
- 每位独立查表或用逻辑电路映射
- 整体 $O(1)$ 延迟（无进位传播）

## 3.7 支持函数（Support Functions）

### 零检测

- **非冗余数制**：零当且仅当所有位为 0（OR 门树，$O(\log k)$ 延迟）
- **冗余数制**：零检测困难！同一数值可能有多种表示（包括非全零的表示），必须先做数字集转换再检测，或使用特殊协议

**问题**：冗余数制无法在 $O(1)$ 时间内判断数值是否为零——这是与非冗余数制的重要区别。

### 符号测试

- **非冗余 2's complement**：只需读最高位（MSB）
- **冗余数制**：符号不确定！同一数可有正、零、负多种等价表示，必须先完成完整加法（归约到非冗余表示）再判符号

### 溢出处理

加法结果超出表示范围：

- **标准检测**：对冗余数制，溢出检测同样困难
- **饱和运算**：溢出时将结果截断到最大/最小表示值（DSP 常用）
- **位宽扩展**：在运算前增加符号扩展位，延迟溢出问题

### 比较运算

`x > y` 等价于 `x - y > 0`（符号测试），因此比较同样困难——需先转换为非冗余表示。

## 3.8 冗余数制的加法器结构总结

```
操作数（冗余 GSD 格式）
         ↓
   ┌─────────────┐
   │ 无进位加法  │  ← 每位独立计算（O(1) 延迟）
   │ （或两步有  │    适用于 ρ≥1
   │  限进位）   │
   └─────────────┘
         ↓
   冗余 GSD 结果（可继续参与后续无进位运算）
         ↓
   ┌─────────────┐（最终需要输出时）
   │ 数字集转换  │  → 非冗余（2's complement 等）
   └─────────────┘
         ↓
   标准二进制结果
```

**延迟分析**：
- 无进位加法：$O(1)$（与位宽 k 无关）
- 最终数字集转换：$O(\log k)$（需要一次超前进位加法）
- 多操作数累加（n 个数）：$O(1)$（冗余加法）+ $O(\log k)$（最终转换），而非 $O(n \log k)$

## 本章小结

```
冗余数制的核心价值
    └─ 用"多余表示"换取"无进位传播加法"

主要格式
    ├── 存储进位（Carry-Save）：数字集 {0,1,2}，r=2，ρ=1
    │   └─ 应用：Wallace Tree 乘法
    ├── BSD：数字集 {-1,0,1}，r=2，ρ=1
    │   └─ 应用：Booth 编码乘法
    └── 高冗余 GSD：r>2 或 ρ≥2
        └─ 应用：CORDIC、高速除法

加法分类
    ├── 无进位加法（r>2，ρ≥1）：$O(1)$，纯组合逻辑
    └── 有限进位加法（r=2，ρ=1）：两步，$O(1)$

代价与限制
    ├── 每位需要更多存储（BSD：2bit/位）
    ├── 零检测、符号测试、比较均困难
    └── 最终输出需转换为非冗余表示（$O(\log k)$）
```

**历史意义**：冗余数制的无进位加法是第 9–16 章高速乘法、除法、平方根和超越函数算法的基础构件。
