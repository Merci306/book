## 第二章 编程模型

## 2.1 高度多线程协处理器

GPU 是一种能够并行执行大量线程的计算设备。它作为主 CPU（即宿主，host）的协处理器运行：换言之，应用程序中运行在宿主上的数据并行、计算密集型部分会被卸载（off-load）到设备（device）上执行。更准确地说，应用程序中需要多次执行、但对不同数据独立运行的部分，可以被抽离为一个**内核函数（kernel function）**，以大量不同线程的形式在 GPU 上执行。为此，该函数被编译为 PTX 指令集，所生成的内核在安装时被翻译为目标 GPU 指令集。

---

## 2.2 线程层次结构（Thread Hierarchy）

执行内核的一批线程被组织为一个**网格（grid）**。如本节所述，网格由协作线程数组（CTA）或协作线程数组的簇（cluster）组成，分别如图 1 和图 2 所示。

- **协作线程数组（Cooperative Thread Arrays，CTA）** 实现 CUDA 线程块（thread block）。
- **簇（cluster）** 实现 CUDA 线程块簇（thread block cluster）。

### 2.2.1 协作线程数组（Cooperative Thread Arrays，CTA）

**PTX（Parallel Thread Execution，并行线程执行）** 编程模型是显式并行的：一个 PTX 程序描述并行线程数组中某一特定线程的执行过程。

**协作线程数组（CTA，cooperative thread array）** 是一组并发或并行执行内核的线程集合。CTA 内的线程可以相互通信。为协调 CTA 内线程之间的通信，可以指定**同步点（synchronization point）**，让线程等待直到 CTA 中所有线程均到达该点。

每个线程在 CTA 内有唯一的**线程标识符（thread identifier）**。程序通过数据并行分解来将输入、工作和结果分配给 CTA 中的各线程。每个 CTA 线程使用其线程标识符来确定自身角色、分配具体的输入输出位置、计算地址、选择要执行的任务。

线程标识符是一个三元向量 `tid`（包含元素 `tid.x`、`tid.y`、`tid.z`），用于指定线程在一维、二维或三维 CTA 中的位置。每个线程标识符分量的范围从零到该 CTA 维度中的线程数。

每个 CTA 的形状由三元向量 `ntid`（包含元素 `ntid.x`、`ntid.y`、`ntid.z`）指定，该向量描述各 CTA 维度上的线程数量。

CTA 内的线程以 **SIMT（单指令多线程，single-instruction, multiple-thread）** 方式执行，并被分组为称为 **warp** 的单元。**Warp** 是来自同一 CTA 的线程的最大子集，其中的线程在同一时刻执行相同的指令。Warp 内的线程按顺序编号。Warp 的大小是与机器相关的常量，通常为 32 个线程。

某些应用可能通过了解 warp 大小来最大化性能，因此 PTX 提供了一个运行时立即常量 `WARP_SZ`，可在允许立即操作数的任何指令中使用。

### 2.2.2 协作线程数组的簇（Cluster of Cooperative Thread Arrays）

**簇（Cluster）** 是一组并发或并行运行的 CTA，它们可以通过共享内存（shared memory）相互同步和通信。执行中的 CTA 在通过共享内存与对等 CTA 通信前，必须确保对等 CTA 的共享内存已存在，且对等 CTA 在共享内存操作完成前尚未退出。

簇内不同 CTA 中的线程可以通过共享内存相互同步和通信。**簇级屏障（cluster-wide barrier）** 可用于同步簇内所有线程。

簇内每个 CTA 在其簇中有唯一的 CTA 标识符（`cluster_ctaid`）。每个 CTA 簇的形状由参数 `cluster_nctaid` 描述，为一维、二维或三维。每个 CTA 在簇中也有跨所有维度的唯一 CTA 排名标识符（`cluster_ctarank`）。簇中所有维度的 CTA 总数由 `cluster_nctarank` 指定。

线程可以通过以下预定义只读特殊寄存器读取并使用这些值：
`%cluster_ctaid`、`%cluster_nctaid`、`%cluster_ctarank`、`%cluster_nctarank`。

簇级别仅适用于目标架构 `sm_90` 或更高版本。在启动时指定簇维度是可选的：
- 若用户在启动时指定了簇维度，则视为**显式簇启动（explicit cluster launch）**；
- 否则视为**隐式簇启动（implicit cluster launch）**，默认维度为 1x1x1。

PTX 提供只读特殊寄存器 `%is_explicit_cluster` 以区分显式和隐式簇启动。

### 2.2.3 簇的网格（Grid of Clusters）

CTA 所能容纳的线程数量有上限，簇所能容纳的 CTA 数量也有上限。但是，执行相同内核的多个簇可以被批量组合为一个**簇网格（grid of clusters）**，使得单次内核调用可启动非常大量的线程。代价是线程间通信和同步能力的降低——不同簇中的线程无法相互通信和同步。

每个簇在簇网格中有唯一的簇标识符（`clusterid`）。每个簇网格的形状由参数 `nclusterid` 描述，为一维、二维或三维。每个网格还有唯一的时序网格标识符（`gridid`）。

线程可以通过以下预定义只读特殊寄存器读取并使用这些值：
`%tid`、`%ntid`、`%clusterid`、`%nclusterid`、`%gridid`。

每个 CTA 在网格中有唯一的标识符（`ctaid`）。每个 CTA 网格的形状由参数 `nctaid` 描述，为一维、二维或三维。线程可通过预定义只读特殊寄存器 `%ctaid` 和 `%nctaid` 读取并使用这些值。

每个内核以线程批次的形式执行，这些线程组织为由 CTA 构成的簇网格，其中簇为可选层级，仅适用于目标架构 `sm_90` 及更高版本。图 1 展示了由 CTA 构成的网格，图 2 展示了由簇构成的网格。

网格之间可以携带依赖关系启动——一个网格可以是某个依赖网格，也可以是某个前置网格。关于如何定义网格依赖关系，请参阅《CUDA 编程指南》中关于 **CUDA Graphs** 的章节。

## 2.3 内存层次结构（Memory Hierarchy）

PTX 线程在执行过程中可以访问多个**状态空间（state space）**中的数据，如图 3 所示（簇层级从目标架构 `sm_90` 起引入）。

各级内存层次结构如下：

| 内存类型 | 可见范围 | 读写权限 | 生命周期 |
|---|---|---|---|
| **私有本地内存（local memory）** | 单个线程 | 读/写 | 线程生命周期 |
| **共享内存（shared memory）** | 同一线程块（CTA）内所有线程，以及簇中所有活跃块 | 读/写 | 与线程块相同 |
| **全局内存（global memory）** | 所有线程 | 读/写 | 内核调用期间及跨调用持久 |
| **常量内存（constant memory）** | 所有线程 | 只读 | 跨内核启动持久 |
| **参数内存（param memory）** | 所有线程 | 只读 | — |
| **纹理内存（texture memory）** | 所有线程 | 只读（带缓存） | 跨内核启动持久 |
| **表面内存（surface memory）** | 所有线程 | 读/写（带缓存） | 跨内核启动持久 |

全局内存、常量内存、参数内存、纹理内存和表面内存针对不同的内存使用场景进行了优化。例如，**纹理内存**提供不同的寻址模式以及针对特定数据格式的数据过滤功能。

> **注意：** 纹理内存和表面内存是带缓存的。在同一内核调用中，缓存不与全局内存写入和表面内存写入保持一致性。因此，对于在同一内核调用中已通过全局写或表面写更新过的地址，纹理读取或表面读取将返回未定义数据。
>
> 换言之，线程只有在目标内存位置由**上一次内核调用或内存拷贝**更新时，才能安全读取纹理或表面内存；若该位置由**同一内核调用中的该线程或其他线程**更新，则不安全。

全局内存、常量内存和纹理内存在同一应用的多次内核启动之间持久存在。

**簇（cluster）** 是一组 CTA 的集合，而 CTA 是执行同一内核程序的并发线程集合。**网格（grid）** 是由独立执行的 CTA 组成的簇集合。

宿主（host）和设备（device）各自维护独立的本地内存，分别称为**宿主内存（host memory）**和**设备内存（device memory）**。设备内存可以被宿主映射并进行读写，或者通过利用设备高性能 **DMA（直接内存访问，Direct Memory Access）** 引擎的优化 API 调用，从宿主内存高效复制数据。
