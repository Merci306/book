# 第 6 章：指令操作数

## 6.1 操作数类型信息

所有指令的操作数都有从声明中确定的已知类型。每个操作数类型必须与指令模板和指令类型所确定的类型兼容。**类型之间不会自动转换**。

- **位大小类型（bit-size type）** 与所有相同大小的类型兼容
- **相同大小的整数类型**彼此兼容
- 与指令类型不同但兼容的操作数，会被**静默强制转换**为指令类型

---

## 6.2 源操作数

源操作数在指令描述中用 **a**、**b**、**c** 表示。

PTX 描述的是**load-store 架构**：ALU 指令的所有操作数必须位于 `.reg` 寄存器状态空间中。大多数操作要求操作数大小一致。

- `cvt`（转换）指令例外，它专门负责在几乎任意数据类型之间转换
- `ld`、`st`、`mov`、`cvt` 负责在状态空间和寄存器之间搬运数据

大多数指令有可选的**谓词守卫（predicate guard）**控制条件执行，谓词操作数用 **p**、**q**、**r**、**s** 表示。

---

## 6.3 目标操作数

产生单一结果的 PTX 指令将结果写入 **d**（destination）字段。结果操作数是寄存器状态空间中的标量或向量变量。

---

## 6.4 地址、数组与向量的使用

### 6.4.1 地址作为操作数

所有内存指令都需要一个地址操作数，支持以下形式：

| 形式 | 含义 |
|---|---|
| `[var]` | 可寻址变量名 |
| `[reg]` | 存有字节地址的整数/位大小寄存器 |
| `[reg+immOff]` | 寄存器地址 + 有符号 32 位常量偏移 |
| `[var+immOff]` | 变量地址 + 有符号 32 位常量偏移 |
| `[immAddr]` | 无符号 32 位立即绝对地址 |
| `var[immOff]` | 数组元素 |

**对齐要求**：地址必须是访问大小的整数倍自然对齐，否则行为未定义。
例：`ld.v4.b32` 访问大小为 16 字节；`atom.f16x2` 访问大小为 4 字节。

**地址宽度**：支持 32 位或 64 位，不支持 128 位。地址按需零扩展，或在超出目标架构状态空间宽度时截断。

所有地址计算均以**字节为单位**，不支持 C 风格的指针算术。

```ptx
.shared  .u16  x;
.reg     .u16  r0;
.global  .v4 .f32  V;
.reg     .v4 .f32  W;
.const   .s32  tbl[256];
.reg     .b32  p;
.reg     .s32  q;

ld.shared.u16      r0, [x];
ld.global.v4.f32   W,  [V];
ld.const.s32       q,  [tbl+12];
mov.u32            p,  tbl;
```

#### 6.4.1.1 通用寻址（Generic Addressing）

若内存指令**不指定状态空间**，则使用通用寻址。

`.const`、kernel `.param`、`.local`、`.shared` 被建模为通用地址空间中的**窗口（window）**，每个窗口由基地址和对应状态空间大小定义：

- 通用地址默认映射到 **global** 内存
- 若地址落在某个窗口内，则映射到对应状态空间（减去窗口基地址）
- kernel `.param` 窗口包含在 `.global` 窗口内

---

### 6.4.2 数组作为操作数

数组标识符成为所在状态空间的**地址常量**，大小是编译期常量。

数组元素访问方式：
- 显式计算字节地址
- 方括号索引：常量整数、寄存器变量、或 `寄存器 ± 常量`

更复杂的索引需要先显式计算地址：

```ptx
ld.global.u32  s, a[0];
ld.global.u32  s, a[N-1];
mov.u32        s, a[1];    // 将 a[1] 的地址存入 s
```

---

### 6.4.3 向量作为操作数

向量可用作源操作数和目标操作数。作为目标时，所有元素必须唯一，否则行为未定义。

用后缀提取向量元素：`.x`、`.y`、`.z`、`.w`（也可用 `.r`、`.g`、`.b`、`.a`）。

花括号列表用于解包向量：

```ptx
.reg .v4 .f32  V;
.reg .f32  a, b, c, d;
mov.v4.f32  {a,b,c,d}, V;
```

元素对应关系：

| 寄存器 | 对应别名 |
|---|---|
| `Ra` | `V.x` = `V.r` |
| `Rb` | `V.y` = `V.g` |
| `Rc` | `V.z` = `V.b` |
| `Rd` | `V.w` = `V.a` |

向量 load/store 实现**宽访问**，可提升内存性能：

```ptx
ld.global.v4.f32  {a,b,c,d}, [addr+16];
ld.global.v2.u32  V2,        [addr+8];
```

---

### 6.4.4 标签与函数名作为操作数

- **标签** — 只能用于 `bra` / `brx.idx` 指令
- **函数名** — 只能用于 `call` 指令
- `mov` 可将函数地址载入寄存器，用于**间接调用**

自 PTX ISA 3.1 起，`mov` 还可获取 **kernel 函数地址**，传递给 GPU 端系统调用以发起 kernel 启动（CUDA Dynamic Parallelism）。

---

## 6.5 类型转换

所有算术、逻辑和数据移动指令的操作数必须**类型和大小相同**（除非改变类型/大小本身就是指令定义的一部分）。不同类型/大小的操作数必须先用 `cvt` 转换。

### 6.5.1 标量转换

**转换操作类型说明：**

| 操作 | 含义 |
|---|---|
| `sext` | 符号扩展 |
| `zext` | 零扩展 |
| `chop` | 截断，只保留低位 |
| `s2f` | 有符号整数 → 浮点 |
| `u2f` | 无符号整数 → 浮点 |
| `f2s` | 浮点 → 有符号整数 |
| `f2u` | 浮点 → 无符号整数 |
| `f2f` | 浮点 → 浮点 |

> **注意**：若目标寄存器比目标格式宽，截断后会根据目标格式进行扩展（符号扩展或零扩展）。
> 例：`cvt.s16.u32` 在 32 位寄存器上：先截断为 16 位，再**符号扩展**至 32 位。

浮点互转支持的格式：`f16`、`f32`、`f64`、`bf16`、`e4m3`、`e5m2`、`e2m3`、`e3m2`、`e2m1`、`ue8m0`、`s2f6`（详见规范完整表格）。

---

### 6.5.2 舍入修饰符

**浮点舍入修饰符：**

| 修饰符 | 含义 |
|---|---|
| `.rn` | 就近舍入（round to nearest even） |
| `.rna` | 就近舍入，同距时远离零 |
| `.rz` | 向零截断 |
| `.rm` | 向负无穷舍入 |
| `.rp` | 向正无穷舍入 |
| `.rs` | 随机舍入（stochastic rounding） |

**整数舍入修饰符：**

| 修饰符 | 含义 |
|---|---|
| `.rni` | 就近取整，同距取偶数 |
| `.rzi` | 向零取整 |
| `.rmi` | 向负无穷取整 |
| `.rpi` | 向正无穷取整 |

---

## 6.6 操作数开销

不同状态空间的操作数访问速度差异显著。**寄存器最快，全局内存最慢。**

隐藏延迟的方法：
1. **多线程** — 硬件发出内存操作后切换执行其他线程
2. **提前发射** — 尽早发出 load 指令；执行只在真正**使用**结果时才会阻塞
