## 3.1 SIMT 多处理器阵列（A Set of SIMT Multiprocessors）

NVIDIA GPU 架构由一组可扩展的多线程**流式多处理器（Streaming Multiprocessors，SM）**组成。当宿主程序（host program）调用一个 kernel grid 时，grid 中的各个线程块（thread block）被枚举并分配给具有空闲执行资源的多处理器。一个线程块中的所有线程在同一多处理器上并发执行。当某个线程块执行完毕后，新的线程块会被调度到该多处理器上继续执行。

多处理器由多个**标量处理器（Scalar Processor，SP）**核心、一个多线程指令单元以及片上共享内存组成。多处理器以零调度开销在硬件层面创建、管理和执行并发线程，并实现了单指令屏障同步（single-instruction barrier synchronization）。

快速屏障同步、轻量级线程创建和零开销线程调度共同支持非常细粒度的并行性——例如可以将一个线程分配给每个数据元素（如图像中的像素、体积中的体素、基于网格计算中的单元格）。

### SIMT 架构

为了管理运行多个不同程序的数百个线程，多处理器采用了一种称为 **SIMT（单指令多线程，Single-Instruction, Multiple-Thread）** 的架构：

- 多处理器将每个线程映射到一个标量处理器核心。
- 每个标量线程独立执行，拥有自己的指令地址和寄存器状态。
- SIMT 单元将线程组织为称为 **warp** 的并行线程组进行创建、管理、调度和执行。（"warp"一词来源于纺织业，是最早的并行线程技术。）

组成同一 SIMT warp 的各个线程从同一程序地址开始执行，但之后可以自由地分支并独立执行。当多处理器被分配一个或多个线程块时，它将其拆分为 warp 并由 SIMT 单元进行调度。

线程块拆分为 warp 的方式始终相同：每个 warp 包含具有连续递增线程 ID 的线程，第一个 warp 包含线程 0。

### Warp 执行与分支散列（Branch Divergence）

在每个指令发射时刻，SIMT 单元选择一个就绪的 warp 并向其活跃线程发出下一条指令。一个 warp 每次执行一条公共指令，因此当 warp 中所有线程的执行路径一致时效率最高。

若 warp 中的线程因数据相关的条件分支而**散列（diverge）**，则该 warp 会串行执行每条分支路径（禁用不在该路径上的线程），所有路径执行完毕后，线程再次**汇聚（converge）**到同一执行路径。

分支散列只发生在同一 warp 内；不同 warp 之间始终独立执行，无论它们执行的代码路径是否相同。

### SIMT 与 SIMD 的对比

SIMT 架构与 SIMD（单指令多数据，Single Instruction, Multiple Data）向量组织类似，都是用单条指令控制多个处理元素。关键区别在于：

- **SIMD** 向量组织将 SIMD 宽度暴露给软件。
- **SIMT** 指令描述单个线程的执行和分支行为。

与 SIMD 向量机相比，SIMT 允许程序员为独立的标量线程编写线程级并行代码，同时也支持为协作线程编写数据并行代码。

从**正确性**角度，程序员基本可以忽略 SIMT 行为；然而，通过注意减少 warp 内线程的散列，可以获得显著的性能提升。这类似于传统代码中缓存行（cache line）的作用：设计正确性时可以安全地忽略缓存行大小，但在设计峰值性能时必须考虑它。

另一方面，向量架构要求软件将 load 合并为向量并手动管理散列。

### 多处理器资源限制

多处理器能同时处理多少线程块，取决于给定 kernel 中每个线程所需的寄存器数量以及每个线程块所需的共享内存大小，因为多处理器的寄存器和共享内存在所有线程之间分配。如果每个多处理器没有足够的寄存器或共享内存来处理至少一个线程块，kernel 将无法启动。

## 3.2 独立线程调度（Independent Thread Scheduling）

在 Volta 架构之前，warp 使用一个由 32 个线程共享的程序计数器，加上一个指定 warp 活跃线程的活跃掩码（active mask）。

这导致：
- 来自同一 warp 的处于散列区域或不同执行状态的线程无法互相发出信号或交换数据。
- 使用锁或互斥量保护数据的细粒度共享算法，容易发生死锁（deadlock），具体取决于竞争线程所在的 warp。

从 **Volta 架构**开始，**独立线程调度（Independent Thread Scheduling）**允许线程之间无论 warp 归属如何都能实现完全并发。GPU 维护每个线程的执行状态，包括程序计数器和调用栈，并可以以线程粒度暂停执行，以便更好地利用执行资源或允许一个线程等待另一个线程产生数据。

一个调度优化器（schedule optimizer）决定如何将来自同一 warp 的活跃线程分组到 SIMT 单元中。这保留了与以前 NVIDIA GPU 相同的高吞吐量 SIMT 执行，但具有更大的灵活性：线程现在可以以子 warp 粒度散列和汇聚。

> **注意**：如果开发者对先前硬件架构的 warp 同步性做了假设，独立线程调度可能导致参与执行代码的线程集合与预期有所不同。特别是任何 warp 同步代码（如无同步的 warp 内规约）都应重新审视以确保与 Volta 及更高版本的兼容性。详情请参阅《CUDA 编程指南》中关于计算能力 7.x 的章节。

## 3.3 片上共享内存（On-chip Shared Memory）

如图 4 所示，每个多处理器拥有以下四种类型的片上内存：

| 内存类型 | 描述 |
|---|---|
| **寄存器（registers）** | 每个处理器一组本地 32 位寄存器 |
| **共享内存（shared memory）** | 所有标量处理器核心共享的并行数据缓存，共享内存空间驻留于此 |
| **常量缓存（constant cache）** | 所有标量处理器核心共享的只读缓存，加速对常量内存空间（设备内存的只读区域）的读取 |
| **纹理缓存（texture cache）** | 所有标量处理器核心共享的只读缓存，加速对纹理内存空间（设备内存的只读区域）的读取；每个多处理器通过实现各种寻址模式和数据过滤功能的**纹理单元（texture unit）**访问纹理缓存 |

本地内存（local memory）和全局内存（global memory）空间是设备内存的可读写区域。
